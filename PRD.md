# 初音未来介绍网页 - 产品需求文档 (PRD)

> 状态：开发中

## 一、产品概述

### 1.1 核心目标
创建永久二维码，扫码后直接打开一个**极具呼吸感和互动性**的初音未来介绍网页。

### 1.2 关键约束（红线）
| 约束 | 要求 |
|------|------|
| 🚫 不需要 VPN | 国内用户直接访问 |
| 🚫 不需要域名 | 用户无需购买域名 |
| ✅ 永久有效 | 二维码一旦印刷，永不过期 |
| ✅ 秒开体验 | CDN加速，1-2秒内加载完毕 |
| ✅ 扫码即开 | 无中间页、无关注步骤 |

### 1.3 目标用户
- 初音未来的粉丝（国内外 fans）
- 动漫/虚拟偶像爱好者
- 偶然扫码的普通用户

## 二、视觉与交互规范

### 2.1 视觉主基调：赛博微光与毛玻璃（Cyber-Glass）

| 元素 | 规范 |
|------|------|
| **主色调** | 初音绿 `#39C5BB` |
| **辅助色** | 纯白 `#FFFFFF`、深灰 `#1E293B` |
| **背景** | 深色系（`#020617` 或更深） |
| **毛玻璃** | `backdrop-filter: blur(20px)` + 半透明背景 |
| **留白** | 大量留白，避免拥挤 |
| **边框** | 抛弃复杂边框，使用柔和阴影 |

### 2.2 图标规范
- 所有图标使用**内联 SVG**
- 不使用 emoji 或外部图标库
- 统一 stroke-width 和颜色

### 2.3 图片规范
| 项目 | 规范 |
|------|------|
| 格式 | **WebP**（需从现有 JPG/PNG 转换） |
| 单张大小 | 150KB - 200KB |
| 加载策略 | 骨架屏 → 懒加载 |
| 响应式 | `srcset` + `object-fit: cover` |

## 三、四大互动模块详细设计

### 模块 A：首屏·诞生与共鸣（Hero Section）

**内容：**
- 标题：初音未来 / Hatsune Miku
- 副标题：2007年8月31日发售，世界上第一位使用全息投影技术举办演唱会的虚拟偶像
- 最高清初音大图（WebP 格式）

**交互设计：**

| 交互 | 实现方式 |
|------|----------|
| 景深视差（Parallax） | 上下滚动时，图片与背景以不同速度移动 |
| 播放按钮 | Web Audio API 渐入经典初音清唱音轨 |
| 音乐淡入 | 音量从 0 渐变到 0.5，时长 2 秒 |

**技术实现：**
- CSS `transform: translateY()` 联动 scroll
- Web Audio API `GainNode` 控制音量

---

### 模块 B：档案·音与形（Profile Section）

**内容：**
| 属性 | 值 |
|------|-----|
| 年龄 | 16 岁 |
| 身高 | 158cm |
| 体重 | 42kg |
| 声源 | 藤田咲 |
| 标志 | 左臂"01" |

**交互设计：**

| 交互 | 实现方式 |
|------|----------|
| 打字机效果 | Intersection Observer 触发，逐字显示档案数据 |
| 字体 | 极简非衬线字体（如 Noto Sans SC） |
| 触摸"01" | 触发青绿色水波纹扩散动画 |

**技术实现：**
- `IntersectionObserver` 监听进入视口
- CSS `@keyframes` 水波纹动画
- `requestAnimationFrame` 打字机效果

---

### 模块 C：画廊·同人与演进（Gallery Section）

**内容：**
- 调用 PIC 目录下的 7 张图片
- 说明：初音形象在创作者笔下千变万化

**交互设计：**

| 交互 | 实现方式 |
|------|----------|
| 横向滑动 | Touch Events 监听，实现沉浸式左右滑动 |
| 骨架屏 | 加载前显示灰色占位骨架 |
| 长按浮起 | Z轴阴影 + 放大效果，松手还原 |
| 灯箱浏览 | 点击查看大图，左右滑动切换 |

**技术实现：**
- `touchstart` / `touchmove` / `touchend`
- CSS `transform: scale() + box-shadow`
- 自定义灯箱组件

---

### 模块 D：彩蛋·交互粒子（全屏隐形互动）

**内容：**
- 整个网页底层绘制青色光点粒子

**交互设计：**

| 交互 | 实现方式 |
|------|----------|
| 粒子跟随 | 手指滑动时，光点受引力聚拢 |
| 持续漂浮 | 无人操作时，粒子缓慢自由漂浮 |
| 极少数量 | 数量控制在 15-20 个，避免性能问题 |

**技术实现：**
- HTML5 Canvas 绘制
- 手指坐标追踪
- 粒子物理模拟（速度 + 引力）

## 四、技术栈（全链路闭环）

### 4.1 前端视觉与交互层

| 层级 | 技术 | 说明 |
|------|------|------|
| 结构 | HTML5 | 语义化标签，无冗余嵌套 |
| 样式 | CSS3 | Flexbox + Grid + CSS Variables + Keyframes |
| 毛玻璃 | `backdrop-filter` | blur + 半透明 |
| 逻辑 | Vanilla JS | 原生 JavaScript，无框架 |
| 滚动监听 | Intersection Observer API | 触发淡入、打字机效果 |
| 触摸 | Touch Events | 画廊滑动、粒子跟随 |
| 音频 | Web Audio API + HTML5 Audio | 背景音乐播放与淡入淡出 |
| 粒子 | HTML5 Canvas | 底层互动粒子系统 |
| 图标 | 内联 SVG | 零网络请求 |

### 4.2 资源优化层

| 资源 | 格式 | 压缩目标 |
|------|------|----------|
| 图片 | WebP | 单张 150-200KB |
| 音频 | MP3 (128kbps) | 约 30 秒片段 |
| 图标 | SVG 内联 | 无额外请求 |

### 4.3 部署与托管层

| 方案 | 优先级 | 说明 |
|------|--------|------|
| **阿里云 uniCloud** | P0 | 国内 CDN，秒开，微信扫码 1-2 秒 |
| **Gitee Pages** | P1 | 备用，国内访问良好 |
| **Cloudflare Pages** | P2 | 国际线路佳，国内需 VPN |

### 4.4 路由与物理入口层

| 项目 | 技术 | 说明 |
|------|------|------|
| 二维码 | **草料二维码**（活码） | 印刷的是活码图案，链接可在后台随时更新 |
| 永久性 | 草料中转 | 即使网址变更，二维码永远有效 |
